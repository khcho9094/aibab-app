# AI 기반 스마트 냉장고 앱 기획서

## 1. 서비스 개요

**서비스명:** 아이밥
**형태:** 웹앱 (모바일 퍼스트)
**한 줄 설명:** 냉장고 식재료를 등록하면 아이 월령에 맞는 이유식/유아식 레시피를 추천하고, 식재료 낭비를 줄여주는 AI 기반 서비스

## 2. 타겟 사용자

- 신생아 ~ 유아를 키우는 가정 (주 사용자: 엄마)
- 이유식/유아식을 어려워하는 초보 부모
- 식재료를 사놓고 버리는 경험이 잦은 육아 가정

## 3. 핵심 가치

1. **식재료 낭비 감소** – 유통기한 임박 재료를 먼저 활용하는 레시피 우선 추천
2. **이유식/유아식 진입 장벽 낮추기** – 아이 월령에 맞는 간단한 레시피 제공
3. **알레르기 안전** – 아이별 알레르기 재료 등록 → 해당 재료 포함 레시피 자동 제외

## 4. MVP 기능 범위

### 포함

| 기능              | 설명                                                                                              |
| ----------------- | ------------------------------------------------------------------------------------------------- |
| 회원가입 / 로그인 | 이메일 기반 인증                                                                                  |
| 아이 프로필       | 이름, 생년월일(→ 월령 자동 계산), 알레르기 등록, 첫 시도 식재료 기록                              |
| 식재료 관리       | 바코드 스캔 또는 수동 입력, 유통기한, 카테고리 CRUD, 자주 쓰는 재료 즐겨찾기, 빠른 연속 입력 모드 |
| 레시피 추천       | 월령 + 보유 재료 기반, 유통기한 임박 재료 우선                                                    |
| 유통기한 알림     | 3일 이내 임박 재료 대시보드 상단 노출                                                             |

> **바코드 스캔을 MVP에 포함하는 이유:** 수동 입력만으로는 육아 중 사용자가 첫 주에 이탈할 가능성이 높음. 바코드 스캔은 구현 난이도가 낮고(오픈소스 라이브러리 활용) 온보딩 마찰을 크게 줄임.

> **바코드 스캔 구현 방식:**
>
> - 카메라 → 바코드 번호(EAN-13) 추출: `html5-qrcode` 또는 `@zxing/browser` 라이브러리
> - 식품 정보 조회: 식품의약품안전처 식품안전나라 API (공공데이터포털, 무료)
> - 바코드로 조회 가능한 정보: 상품명, 원재료, 제조사
> - **유통기한은 바코드에 없음** → 스캔 후 사용자가 직접 입력 필요 (포장지에 별도 인쇄)
>
> **실제 UX:** 바코드 스캔 → 상품명 자동 입력 → 유통기한만 수동 입력 → 등록 완료

> **바코드 조회 실패 케이스 (소규모 브랜드, 수입품):** 공공 API DB에 없는 상품 비율이 높음 → "상품 정보를 찾을 수 없습니다. 이름을 직접 입력해주세요" 화면으로 graceful fallback 처리 필수

> **식재료 입력 마찰 감소 UX (MVP 포함):**
>
> - **즐겨찾기:** 자주 등록하는 재료(당근, 달걀 등)를 즐겨찾기 저장 → 탭 한 번으로 수량/기한만 바꿔 재등록
> - **빠른 연속 입력 모드:** 한 화면에서 여러 재료를 연속 입력하여 장을 본 직후 한 번에 등록 가능

### MVP 이후 (v2+)

- 영수증 OCR
- 카메라 AI 식재료 인식
- 쇼핑몰 연동 (쿠팡, 마켓컬리 등)
- 푸시 알림 (PWA)
- 커뮤니티 / 레시피 공유
- 정기 배송 구독 연동

## 5. 핵심 사용자 플로우

```
[회원가입]
    ↓
[아이 정보 등록] – 생년월일, 알레르기, 첫 시도 재료 기록
    ↓
[식재료 등록] – 바코드 스캔 or 수동 입력, 유통기한
    ↓
[대시보드]
   ├── 유통기한 임박 재료 경고
   └── 오늘의 추천 레시피 (월령 맞춤)
    ↓
[레시피 상세] – 단계별 조리법, 보관 팁, 면책 안내
    ↓
[첫 시도 기록] – 새 재료 먹였을 때 날짜 + 반응 메모
```

## 6. 레시피 추천 로직

```
1. 현재 아이 월령 계산 (생년월일 기준)
2. 월령 범위 내 레시피 필터링
3. 아이 알레르기 재료 포함 레시피 제외
4. 보유 식재료와 레시피 재료 매칭 점수 계산
5. 유통기한 임박 재료 포함 시 점수 보너스
6. 상위 N개 추천 (재료 부족 레시피는 "재료 부족" 표시)
```

## 7. 레시피 데이터 전략 및 검수 프로세스

**방식:** AI 사전 생성 (기본) + 실시간 AI 생성 (보조)

| 방식        | 설명                                                  | 비용                |
| ----------- | ----------------------------------------------------- | ------------------- |
| 사전 생성   | 월령별 × 주요 재료 조합으로 200~500개 생성 후 DB 저장 | 초기 1회, $1~5 수준 |
| 실시간 생성 | DB에 없는 재료 조합 요청 시 AI 호출 → 결과 DB 캐싱    | 요청당 ~$0.001      |

**월령 그룹:**

- 4~6개월 (초기 이유식)
- 6~9개월 (중기 이유식)
- 9~12개월 (후기 이유식)
- 12개월+ (유아식)

**주요 재료 (핵심 20종 예시):**
당근, 감자, 고구마, 애호박, 브로콜리, 시금치, 소고기, 닭고기, 두부, 달걀, 쌀, 오트밀, 사과, 바나나, 배, 단호박, 양파, 무, 연두부, 쌀미음

**주의 사항 (레시피 생성 시 반드시 반영):**

- 꿀 → 12개월 미만 금지
- 생우유 → 12개월 미만 금지
- 견과류 → 알레르기 주의
- 나트륨 → 이유식 단계별 제한

**검수 프로세스:**

1. AI 생성 레시피에 금지 재료 자동 필터링 로직 적용 (코드 레벨)
2. 사전 생성 레시피 전량 사람이 1차 검토 후 DB 적재 → **"검수 완료" 배지** 표시
3. 실시간 생성 레시피는 프롬프트에 월령별 금지 재료 목록 명시 → 생성 후 자동 검증 → **"AI 생성 (베타)" 배지** 표시
4. 레시피 하단에 면책 문구 필수 표시 → "이 레시피는 참고용입니다. 아이의 건강 상태에 따라 소아과 전문의와 상담하세요."

**레시피 신뢰도 배지:**

| 배지         | 조건                          | 표시             |
| ------------ | ----------------------------- | ---------------- |
| ✅ 검수 완료 | 사람이 1차 검토 완료된 레시피 | 초록색 배지      |
| 🤖 AI 생성   | 실시간 생성, 자동 검증만 완료 | 회색 배지 (베타) |

## 8. 기술 스택

| 레이어       | 기술                    | 이유                                         |
| ------------ | ----------------------- | -------------------------------------------- |
| 프론트 + API | Next.js 14 (App Router) | SSR/SEO + API Routes로 풀스택 단일 프로젝트  |
| DB + Auth    | Supabase                | PostgreSQL + 인증 내장, 무료 티어로 MVP 충분 |
| AI           | Claude API (Haiku 4.5)  | 한국어 품질 우수, 저비용 레시피 생성         |
| 배포         | Vercel                  | Next.js 최적화, 무료로 시작 가능             |

**초기 비용:** 사실상 $0 (무료 티어 활용, 레시피 사전 생성 $1~5)

## 9. DB 스키마 (초안)

```sql
-- 아이 프로필
children (
  id uuid PK,
  user_id uuid FK → auth.users,
  name text,
  birthdate date,
  created_at timestamptz
)

-- 알레르기 (세분화: 재료명 + 심각도 + 메모)
child_allergies (
  id uuid PK,
  child_id uuid FK → children,
  ingredient_name text,        -- '달걀 흰자', '새우' 등 세분화
  severity text,               -- 'mild' | 'severe'
  note text,
  confirmed_at date
)

-- 첫 시도 식재료 기록
first_tries (
  id uuid PK,
  child_id uuid FK → children,
  ingredient_name text,
  tried_at date,
  reaction text,                -- '이상 없음', '발진', '구토' 등
  note text
)

-- 재료명 정규화 테이블 (동의어 매핑, 초기 100~200개)
ingredient_aliases (
  alias text PK,               -- '쇠고기', '한우', 'beef'
  canonical_name text          -- '소고기'
)

-- 식재료
ingredients (
  id uuid PK,
  user_id uuid FK,
  name text,                   -- 사용자 입력값 원본
  canonical_name text,         -- 정규화된 재료명 (레시피 매칭에 사용)
  quantity numeric,
  unit text,
  expiry_date date,
  category text,
  is_favorite boolean default false,
  created_at timestamptz
)

-- 레시피
recipes (
  id uuid PK,
  title text,
  description text,
  age_min_months int,
  age_max_months int,
  steps text[],
  tip text,
  is_ai_generated boolean,
  verified_status text default 'ai_beta',  -- 'verified' | 'ai_beta'
  created_at timestamptz
)

-- 레시피-재료 매핑
recipe_ingredients (
  recipe_id uuid FK,
  ingredient_name text,
  quantity text,
  is_optional boolean default false
)

-- 저장한 레시피
saved_recipes (
  user_id uuid FK,
  recipe_id uuid FK,
  saved_at timestamptz
)
```

## 12. 개발 단계 (MVP 기준)

| 단계  | 내용                                                                      |
| ----- | ------------------------------------------------------------------------- |
| 1단계 | 프로젝트 셋업, Supabase 연결, 기본 인증                                   |
| 2단계 | 아이 프로필 CRUD                                                          |
| 3단계 | 식재료 CRUD + 유통기한 관리 + 즐겨찾기/빠른 입력 UX                       |

| 7단계 | 동적 AI 레시피 (베타, 자동 검증 + AI 배지)                                |
| 8단계 | UI 완성 (모바일 퍼스트) + 프리미엄 얼리버드 CTA                           |
